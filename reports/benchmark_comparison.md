# Benchmark 对比报告：CPU Baseline vs GPU (RTX 4050 Laptop)

### CPU 环境（Mac）

- **硬件**: MacBook (Apple Silicon), CPU 推理
- **推理引擎**: Ollama v0.18.2
- **模型**: Qwen2.5-1.5B (986MB)
- **压测工具**: benchctl (Go goroutine 并发客户端)
- **时间**: 2026-03-21

### GPU 环境（Windows）

- **硬件**: Windows 笔记本, NVIDIA GeForce RTX 4050 Laptop GPU (6141 MiB)
- **驱动**: 591.59
- **推理引擎**: Ollama v0.18.1 (CUDA)
- **模型**: Qwen2.5-1.5B (986MB)
- **压测工具**: benchctl (Go goroutine 并发客户端)
- **时间**: 2026-03-22

## 4. 瓶颈分析

### CPU → GPU 加速总结

1. **QPS 翻倍**：GPU 在所有并发级别下 QPS ≈ 2.4，CPU 硬顶 ≈ 1.1。GPU 推理的计算吞吐是 CPU 的 **2x**
2. **P50 延迟减半**：低并发下 GPU P50=1162ms vs CPU 2621ms（-56%），说明单次推理的 decode 速度显著提升
3. **tokens/s 提升 65%**：GPU 在 c=3 下达到 48.65 tok/s vs CPU 的 29.55

### GPU 并发扩展性分析

1. **QPS 未随并发增长**：GPU QPS 在 c=3/10/20 下基本持平（2.36–2.48），说明 Ollama 在 RTX 4050 上仍有串行瓶颈（单 GPU 推理线程）
2. **延迟随并发线性增长**：c=3 P50=1162ms → c=10 P50=4058ms → c=20 P50=7848ms，符合排队模型
3. **每请求 tok/s 下降**：48.65 → 17.84 → 9.92，有效 decode 时间被排队时间稀释

### CPU vs GPU 扩展性差异

| 行为 | CPU (Mac) | GPU (RTX 4050) |
|---|---|---|
| QPS 是否随并发增长 | ❌ 不变（1.13） | ❌ 基本不变（2.36–2.48） |
| c=3→c=10 P50 增幅 | +236%（2621→8812） | +249%（1162→4058） |
| 扩展瓶颈 | CPU 串行推理 | GPU 单推理线程，未利用张量并行 |

## 5. 结论

1. **RTX 4050 Laptop GPU 在推理吞吐上是 Apple Silicon CPU 的 2x**，单次推理延迟减半
2. 两者都存在 **QPS 硬顶**，但硬顶位置不同：CPU ≈ 1.1，GPU ≈ 2.4
3. 进一步提升需要：**多 GPU / 张量并行 / vLLM continuous batching**——Ollama 目前不支持 concurrent request 级并行
4. 对于实际生产部署，应根据单次推理延迟预算反推最大并发：GPU 环境下建议 c ≤ 5（维持 P50 < 2s）
