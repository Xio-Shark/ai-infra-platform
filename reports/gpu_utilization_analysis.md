# GPU Utilization Profile: RTX 4050 Laptop + Ollama Qwen2.5-1.5B

## 测试配置

- **GPU**: NVIDIA GeForce RTX 4050 Laptop GPU (6141 MiB)
- **模型**: Qwen2.5-1.5B (986MB)
- **推理引擎**: Ollama v0.18.1
- **压测**: benchctl c=5, total=15, max_tokens=64
- **采集工具**: `nvidia-smi dmon -s u -d 1 -c 30`

## 分析

### 关键发现

1. **GPU SM 利用率峰值仅 54%**，说明 Ollama 没有充分利用 RTX 4050 的计算能力
2. **高并发阶段（前 9 秒）SM 稳定在 46-54%**，显示 GPU 有 ~50% 算力闲置
3. **MEM 利用率峰值 39%（2.4GB / 6.1GB）**，内存完全不是瓶颈
4. **SM 利用率随并发降低而下降**：c=5 并发耗尽后 SM 掉到 19-38%，说明 Ollama 无法并行处理多个推理请求

### 瓶颈定位

| 资源 | 利用率 | 是否瓶颈 |
|------|--------|---------|
| GPU SM（算力） | 46-54% 峰值 | ❌ 未饱和——Ollama 单线程串行推理，无法填满 SM |
| GPU Memory | 39% 峰值 | ❌ 远未饱和——6GB VRAM 对 1.5B 模型绰绰有余 |
| CPU（推理调度） | — | ✅ 可能——Ollama 的请求调度是 CPU 侧串行 |
| 网络（HTTP） | — | ❌ localhost 无瓶颈 |

### 结论

RTX 4050 Laptop GPU 在 Ollama 下**算力浪费 ~50%**。根因是 Ollama 对并发请求采用串行 decode，无法利用 GPU 的张量并行能力。生产环境应使用 **vLLM / SGLang** 的 continuous batching 来提升 SM 利用率至 80%+，预期 QPS 可进一步提升 2-3x。
